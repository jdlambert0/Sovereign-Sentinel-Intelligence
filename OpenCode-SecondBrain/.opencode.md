OBSIDIAN ARCHIVE PROTOCOL

- Session Logging: End of every task/session, write a log to Sessions/YYYY-MM-DD-<task-name>.md
  - Frontmatter: date, task-id, tags: [#session-log]
  - Content: Summary, Decisions, Changes, and [[WikiLinks]] to relevant Topic notes.
  - See Also: The bottom must include a "## See Also" section with links.

- See Also Rule: All links must be appended to a ## See Also section at the bottom. No inline modification of existing notes.

- Topic Creation: If a project or tool is mentioned 3+ times, prompt to create a new note in Topics/.
- Topic Creation: If a project or tool is mentioned 3+ times, OpenCode should prompt to create a new note in [VAULT_PATH]/Topics/.
- Vault Read Integration: OpenCode should search the vault before invoking external tools. If a past decision or project detail exists in the vault, reuse it.
- Vault Read Implementation Note: The vault search should target Sessions/ and Topics/ primarily, with Resources/ for research notes.

## Vault Read & Integration (Summary)
- Before web searches or directory exploration, OpenCode will grep [VAULT_PATH]/Sessions and [VAULT_PATH]/Topics for relevant context. If a matching note is found, content will be incorporated into the current task context.
